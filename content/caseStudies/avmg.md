## 2. CHALLENGES & INSIGHTS

- **Challenge: Ephemeral Worker Disk Exhaustion**
  During simultaneous multi-gigabyte downloads and conversion runs (e.g., converting heavy video containers to high-resolution MP4s or generating long GIF loops), intermediate file payloads quickly consumed the local disk capacity of the AWS EC2 worker instance, leading to critical disk volume exhaustion.
- **Challenge: Subtitle Scraping Bloat and Overhead**
  Pulling subtitle content from remote media sources using traditional methods forced the download of the entire heavy video binary first, generating significant network latency, CPU cycles, and disk writes, even when the end-user only wanted a lightweight `.txt` transcript file.
- **Challenge: VTT Caption Noise and Subtitle Stuttering**
  Raw `.vtt` captions scraped from automated subtitle tracks were heavily contaminated with alignment parameters (`align:start position:9%`), standard and inline karaoke timestamps (`<00:00:29.119>`), bracketed sound descriptions (`[Music]`), HTML formatting, and multiple duplicate lines generated by dynamic video text scrolling.
- **Challenge: Severe GIF Color Banding & Frame Rendering**
  Standard single-pass GIF transcodes in FFmpeg mapped the wide color range of modern digital videos to a static, low-fictional generic color palette. This resulted in ugly color banding, blocky dithering artifacts, and extremely large output file sizes.
- **Challenge: High-Frequency Database Locking & Webhook Synchronization**
  Rapid, concurrent state updates from distributed workers pushing job telemetry (e.g., transitioning from `PENDING` to `PROCESSING` to `COMPLETED`/`FAILED`) risked race conditions, locks in PostgreSQL, and desynchronization in real-time UI dashboards.

- **Insight: The Ghost Mode Bypass Protocol**
  By intercepting requests where the target format is explicitly `.txt` in [downloader.js](file:///e:/AVMG/b/src/services/downloader.js#L12-L34), the extraction worker bypasses the media binary stream entirely. We pass `skipDownload: true` along with subtitle extraction configs (`writeSubs: true`, `writeAutoSubs: true`, and `subLangs: "en"`) to `youtube-dl-exec`, reducing download weight from hundreds of megabytes to a sub-millisecond network request for the subtitle track alone.
- **Insight: Regex Chains and Stutter Reduction Engine**
  We implemented a deterministic processing pipeline in [transcriptCleaner.js](file:///e:/AVMG/b/src/services/transcriptCleaner.js#L3-L43) that cleans dirty subtitle transcripts. It uses a sequence of regex replacements to clean alignment metadata, karaoke tags, HTML, and audio markers, followed by a linear line-by-line deduplication algorithm that merges consecutive duplicate fragments to produce clean running prose.
- **Insight: Lanczos Scale Split-Palette Generation**
  For `.gif` output mutations, we bypass default transcoding by injecting a complex two-pass FFmpeg filter layout: `fps=15,scale=480:-1:flags=lanczos,split[s0][s1]`; `[s0]palettegen[p]`; `[s1][p]paletteuse` in [mediaProcessor.js](file:///e:/AVMG/b/src/services/mediaProcessor.js#L55-L63). This splits the input stream, dynamically constructs a custom palette representing the specific colors in the source video, and applies the palette during rendering to ensure high-fidelity color output.
- **Insight: Ephemeral Garbage Collection Lifecycle**
  We enforced a zero-leak storage guarantee. Both original uploads and intermediate transcode variants are automatically purged. In [mediaProcessor.js](file:///e:/AVMG/b/src/services/mediaProcessor.js#L78-L92), a filesystem cleanup hook triggers `fs.unlink` in both the completion and rejection handlers. For finalized outputs, [supabaseUplink.js](file:///e:/AVMG/b/src/utils/supabaseUplink.js#L36-L38) triggers a synchronous `fs.unlinkSync` immediately after verifying upload completion to the `avmg-payloads` bucket.
- **Insight: Decoupled Queue Telemetry & State Ledgers**
  Rather than writing to the database directly from API handlers, we leverage Prisma to generate a system receipt in PostgreSQL, offload the heavy execution payload to BullMQ on Redis, and update the telemetry ledger asynchronously in [mediaWorker.js](file:///e:/AVMG/b/src/workers/mediaWorker.js#L40-L43). The frontend app uses a non-blocking 5-second polling interval to read database records without blocking the main event loops.

## 3. OVERVIEW

AVMG provides a distributed, decoupled media mutation service that enables high-efficiency video transcoding, network asset pulls, and silent transcript extraction. Users interact with neon-accented, retro-futuristic CRT dashboard terminals to drag-and-drop local media uploads or designate remote network URLs. Once ingested, the target assets are partitioned by pipeline rules: heavy video payloads are routed to multi-stage FFmpeg processing scripts, while text requests are parsed using subtitle bypass engines. Every finished transcode or cleaned script is beamed to public cloud storage buckets, leaving a telemetry footprint in a central ledger database.

The underlying infrastructure utilizes a Next.js BFF (Backend-for-Frontend) architecture coupled to a distributed Express.js backend. Communication between the API gateway and the worker instances is mediated by a Redis-backed BullMQ message queue, maintaining high availability during sudden traffic spikes. Node.js daemon workers dynamically scale to run compute-intensive tasks on dedicated environments. Telemetry status is monitored through lightweight API gateways and reflected on client dashboards via automated, optimized polling.

## 4. KEY FEATURES

### Local Media Transcoding Engine (FFmpeg Core)
- **Fast-Seeking Optimizations:** Uses input-seeking via `command.seekInput(startTime)` in [mediaProcessor.js](file:///e:/AVMG/b/src/services/mediaProcessor.js#L39-L42), instructing FFmpeg to bypass decoding intermediate frame indexes prior to the offset timestamp, decreasing processing time for long clips.
- **Precision Quality Brackets:** Implements strict compression constraints, enforcing audio bitrates of `320k` (best), `192k` (good), and `128k` (draft), alongside video resolution scaling that clamps outputs to `1280x720` (good) or `854x480` (draft).
- **Watermark Filter Overlays:** Statically binds user logo graphics into video container streams using a complex coordinate overlay filter (`overlay=W-w-10:H-h-10`) positioned relative to width and height boundaries.
- **Snapshot Thumbnail Extraction:** Captures standalone image thumbnails (.jpg or .png) by overriding FFmpeg video stream parameters to write only a single frame output (`-vframes 1`).

### Stealth Subtitle Extractor (Ghost Protocol)
- **Zero-Download Subtitle Scrapes:** Employs target format checking to configure remote resource downloads with `skipDownload = true`, avoiding binary data pulls.
- **Timestamp & Layout Sanitization:** Employs a sequence of regular expressions to clean `.vtt` headers, karaoke sub-timing strings, positional alignment parameters, and non-verbal descriptors.
- **Deduplication Filter:** Resolves scrolling auto-caption duplication via a sequential filter loop, returning polished, paragraph-stitched text logs.

### Asynchronous Distributed Job Queue (BullMQ + Redis)
- **Decoupled Architecture:** Node.js workers run in separate execution contexts to shield main Express web servers from CPU starvation during transcodes.
- **Redis Communication Rail:** Persists job payload specifications (`jobId`, source URLs, target formats, trimming limits) on a durable Redis queue database.
- **Robust Telemetry Updates:** Synchronizes state changes (`PENDING` $\rightarrow$ `PROCESSING` $\rightarrow$ `COMPLETED`/`FAILED`) within a PostgreSQL instance.

### Cloud Uplink Engine (Supabase Storage Integrations)
- **S3-Compatible Object Storage:** Beams mutated files to bucket `avmg-payloads` with upsert policies, outputting highly available download URLs.
- **Storage Protection Policy:** Automatically triggers synchronous filesystem unlink operations to delete temporary assets locally the instant cloud uploading finishes.

### Telemetry Dashboard & Ledger
- **Relational Integrity Logs:** Displays chronological summaries of the fifty most recent processing events.
- **Live Status Badges:** Uses CSS animation animations to represent live states (`COMPLETED`, `FAILED`, `PROCESSING`).
- **Download Authorization Lock:** Locks public retrieval access for uncompleted or crashed jobs.

### Webhook Notification Dispatcher
- **Async Callback System:** Fires HTTP POST payloads back to user-configured webhook target endpoints upon job termination.
- **Structured Payload Format:** Transmits JSON structures representing the execution event details, including the job ID, final status, filename, and secure download link.

## 6. WORKFLOWS

### Ingestion Flow
1. **Client Submission:** The client uploads a local file via Next.js or specifies a remote video URL, target parameters, and optionally a watermark and webhook config.
2. **Gateway Ingestion:** The Express.js API receiver parses media uploads using Multer and saves details to local disks.
3. **Database Receipt Generation:** Prisma registers a `MediaJob` entry with status `PENDING` in PostgreSQL.
4. **BullMQ Dispatch:** The server pushes the parameters (`jobId`, format, quality, offsets) to `MediaProcessingQueue` on Redis.
5. **Fast Gateway Return:** The backend returns the unique `jobId` immediately, letting the client transition to polling.
6. **Worker Intake:** The background daemon worker (`mediaWorker.js`) detects the job, sets the state to `PROCESSING`, and routes it via the execution switchboard.
7. **Asset Processing:**
   - **Local Transcode (`job-convert`):** Triggers FFmpeg seeking, bitrates, watermark composition, and writes the output.
   - **Network Pull (`job-download`):** Invokes `yt-dlp` to download the remote stream.
   - **Combo pipeline (`job-download-convert`):** Downloads the media, runs subtitles parsing, and invokes FFmpeg for final transcodes.
8. **Cloud Uploading:** The local file buffer is uploaded to the Supabase cloud bucket `avmg-payloads`.
9. **Ledger Finalization:** Prisma updates the database status to `COMPLETED` and registers the cloud download URL.
10. **Garbage Collection:** Synchronous unlinking removes local files to prevent disk usage.
11. **Webhook Notification:** A POST event is sent to the target `webhookUrl` if specified.

### Query Flow
1. **Telemetry Request:** The dashboard or polling interface fires an HTTP request (`GET /status/:jobId` or `GET /jobs`).
2. **Ledger Extraction:** Express queries PostgreSQL using Prisma's `findUnique` or `findMany` filters.
3. **Data Marshalling:** Database rows containing job parameters, state, and paths are serialized to JSON.
4. **State Presentation:** The Next.js UI processes the status field. If `PROCESSING`, it loops an animation; if `FAILED`, it throws an alert; if `COMPLETED`, it unlocks the target file download link.

## 7. DATA MODEL

### `User` Table
Tracks user identities and links to job assignments.
*   `id` (String, UUID, Primary Key): Unique identifier.
*   `email` (String, Unique): User authentication address.
*   `name` (String, Nullable): Display name.
*   *Relations:* `mediaJobs` (One-to-many relationship mapping to `MediaJob[]`).

### `MediaJob` Table
Acts as the central telemetry ledger for all media transcodes, network downloads, and transcriptions.
*   `id` (String, UUID, Primary Key): Unique database receipt identifier.
*   `userId` (String, Nullable, Foreign Key): Optional reference mapping the job to a specific `User`.
*   `jobType` (String): Specifies execution paths: `"job-convert"`, `"job-download"`, or `"job-download-convert"`.
*   `originalName` (String, Nullable): Filename of local uploads.
*   `url` (String, Nullable): Remote source target location.
*   `targetFormat` (String, Nullable): Export target format extension (e.g. `mp3`, `wav`, `mp4`, `gif`, `jpg`, `txt`).
*   `quality` (String, Nullable): Compression level profile (`best`, `good`, `draft`).
*   `fileName` (String, Nullable): Final name of the cloud-stored file.
*   `path` (String, Nullable): Permanent Supabase public download URL.
*   `size` (Integer, Nullable): Size in bytes.
*   `transcriptPath` (String, Nullable): Temporary internal route for subtitles.
*   `status` (JobStatus Enum): Flow state, defaults to `PENDING`.
*   `createdAt` (DateTime): Automated creation timestamp.
*   `updatedAt` (DateTime): Automatic modification tracker.

### `JobStatus` Enum
Restricts job states to ensure integrity:
*   `PENDING`: Job registered but waiting in queue.
*   `PROCESSING`: Job currently active in worker.
*   `COMPLETED`: Asset uploaded to cloud storage and local file removed.
*   `FAILED`: Core processing failure.

## 8. OUTCOME

The Audio-Video Mutation Grid (AVMG) provides a highly resilient, production-grade media pipeline. By decoupling network-heavy downloads and resource-intensive FFmpeg transcodes using BullMQ and Redis, the gateway remains responsive. The implementation of the **Ghost Mode Bypass Protocol** eliminates unnecessary network overhead for transcript generation, and the immediate filesystem cleanup ensures consistent disk usage on worker nodes. Backed by PostgreSQL and a robust Next.js frontend console, AVMG provides secure, traceable media processing suitable for enterprise use.
